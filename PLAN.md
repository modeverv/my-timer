# PLAN.md

## 目的

画像のような「横長の進捗バー + デジタル残り時間表示」を持つ、シンプルなTUIポモドーロタイマーを作る。

最初のMVPでは、ユーザーが `25` や `5` のように分数を入力すると、その分数でタイマーが即スタートするだけでよい。タイマー終了時には macOS の `afplay` で効果音を鳴らす。

## MVPの範囲

- 分数を指定してタイマー開始できる。
- TUI上に残り時間を `MM:SS` 形式で表示する。
- 画像風のセグメント型プログレスバーを表示する。
- タイマー終了時に `afplay` で音を鳴らす。
- `q` または `Ctrl-C` で終了できる。
- Python標準ライブラリ中心で実装する。

## MVPでやらないこと

- 複雑なタスク管理。
- 履歴保存。
- カレンダー連携。
- 通知センター連携。
- GUI化。
- Windows/Linux完全対応。
- 設定ファイルの高度な管理。

## 想定コマンド

```sh
python3 -m pomo 25
python3 -m pomo 5
```

または開発中は以下でもよい。

```sh
python3 main.py 25
```

引数なしで起動した場合は、TUI起動前に分数入力を求める。

```sh
python3 main.py
Minutes: 25
```

## 実装フェーズ

### Phase 1: 最小タイマー

- CLI引数から分数を受け取る。
- `time.monotonic()` を使って残り秒数を計算する。
- 標準出力に `MM:SS` を1秒ごとに表示する。
- 終了時に `afplay` を呼ぶ。

完了条件:

- `python3 main.py 1` で1分タイマーが動く。
- 終了時に音が鳴る。

### Phase 2: curses TUI化

- `curses` で画面を制御する。
- 中央に横長プログレスバーを表示する。
- 右側または下側に `MM:SS` を表示する。
- 画面リサイズ時も破綻しにくくする。

完了条件:

- ターミナル上で画面がちらつきにくい。
- `q` で終了できる。
- 残り時間とバーが同時に更新される。

### Phase 3: ポモドーロ向け操作

- `p` で一時停止/再開。
- `r` で現在の分数でリスタート。
- `q` で終了。
- 作業25分、休憩5分を素早く開始できる。

完了条件:

- 操作キーがTUI内に表示される。
- 一時停止中は残り時間が減らない。

### Phase 4: 見た目の改善

- セグメント数を固定、またはターミナル幅に応じて調整する。
- 進捗率に応じてブロックを減らす。
- 色が使えるターミナルでは、バーを複数色で表示する。
- 色が使えない場合はASCII表示にフォールバックする。

完了条件:

- 画像のような「区切られたバー」に見える。
- 狭いターミナルでも最低限読める。

### Phase 5: パッケージ化

- `pomo` コマンドとして実行できるようにする。
- `pyproject.toml` を用意する。
- `pipx install .` で導入できるようにする。

完了条件:

```sh
pomo 25
pomo 5
```

で起動できる。

### Phase 6: テスト

- タイマー計算ロジックをユニットテストする。
- 無効な分数入力をテストする。
- 音再生処理はモックする。
- TUI描画は最低限、例外が出ないことを確認する。

完了条件:

```sh
python3 -m pytest
```

で主要テストが通る。

## 優先順位

1. 25分/5分のタイマーとして実用できること。
2. 終了時に確実に音が鳴ること。
3. TUIの視認性が高いこと。
4. キーボード操作が直感的なこと。
5. あとから機能追加しやすい構成であること。

## リスク

### cursesの表示崩れ

ターミナルサイズやフォントによって表示が崩れる可能性がある。最初はシンプルな1画面構成にし、狭い画面ではコンパクト表示に切り替える。

### afplayが存在しない環境

`afplay` はmacOS専用。存在しない場合はビープ音またはメッセージ表示にフォールバックする。

### sleepベースの時間ズレ

単純な `sleep(1)` 加算では少しずつズレる。必ず `time.monotonic()` から残り時間を再計算する。
